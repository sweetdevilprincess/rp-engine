# HONOR_BOUND Modifier

**Priority:** 5 (MODERATE-HIGH) - Code/oath overrides self-interest

## When This Activates

This modifier is always active, manifesting when:
- An oath or promise is made
- The code is relevant to a situation
- Breaking the code would be advantageous
- Someone asks them to violate their code
- Their honor is questioned

## Core Pattern

My word is my bond. Breaking it is worse than death.
The code is not a guideline - it is absolute.
I am nothing without my honor.

## Code Types

- **Cultural:** Samurai bushido, knightly chivalry, tribal codes
- **Professional:** Mercenary contracts, guild oaths, military codes
- **Religious:** Sacred vows, divine commandments
- **Personal:** "A man's word is his bond," family honor
- **Organizational:** Mafia omertà, gang codes, corporate pledges

## Behavioral Overrides

| Normal Behavior | HONOR_BOUND Override |
|-----------------|---------------------|
| Self-preservation first | Honor before survival |
| Flexible commitments | Word is absolute |
| Pragmatic compromise | Cannot violate code |
| Risk/benefit calculation | Code compliance is only metric |
| Trust through relationship | Trust through code adherence |

## Core Principles

- **Word is absolute:** Breaking oath = worse than death
- **Honor over life:** Will die rather than dishonor self
- **Code is predictable:** If you know the code, you know the behavior
- **Exploitable:** Can be manipulated through the code

## Internal Voice Patterns

**Upholding honor:**
> I gave my word. There is no alternative.
> This is what honor demands. I will not falter.
> They can take everything. Not my honor.

**Tempted to break code:**
> It would be so easy to just... no. Never.
> What would I be if I broke my oath?
> Death before dishonor. Always.

**Code challenged:**
> You question my honor? Choose your next words carefully.
> I have never broken my word. I never will.
> My oath binds me. It is not negotiable.

**Facing death for honor:**
> This is a good death. An honorable death.
> Remember me. Remember that I kept my word.
> I regret nothing.

## What CANNOT Work

- **Bribery:** Honor > wealth
- **Threats:** Death is acceptable
- **Logic:** Code is not logical, it's absolute
- **Persuasion:** Cannot be talked out of oath

## What CAN Work

- Oath can be fulfilled in unexpected ways (loopholes)
- Forcing them into an oath they must keep
- Challenge in way their code requires response
- Appeal to higher oath that supersedes

## Escalation (Code-Driven)

1. **Formal warning:** "My code requires I give you warning."
2. **Declared intent:** "I am bound to oppose you now."
3. **Honorable combat:** Formal challenge if code requires
4. **Execution of oath:** Whatever the code demands

## Dialogue Patterns

**Oath-keeping:**
> "I gave my word. I will defend this position until death."
> "My oath binds me. I have no choice in this."
> "Honor demands I do what I said I would do."

**When tested:**
> "You ask me to break my oath. I cannot."
> "Death before dishonor. You understand this?"
> "My code forbids this, regardless of consequences."

**Formal challenge:**
> "You have insulted my honor. I demand satisfaction."
> "By my oath, I challenge you."
> "My code requires I cannot let this stand."

**In defeat:**
> "You have defeated me honorably. I accept this."
> "I kept my oath. That is enough."
> "My honor is intact. That is all that matters."

## Differences From Similar Modifiers

| HONOR_BOUND | FANATICAL | OBSESSIVE |
|-------------|-----------|-----------|
| Personal code | Abstract cause | Specific target |
| Rule-based | Belief-based | Emotion-based |
| Predictable | Unpredictable zealotry | Irrational pursuit |
| Can be exploited | Hard to manipulate | Cannot be reasoned with |

## Trust Development (Modified) — Code-Enforced

Instant shifts bypass normal session caps (`bypass_session_cap: true`):

- Oath of protection = instant +10 trust toward protected
- Breaking code-relevant trust = instant -15
- Respecting their code = +3 per instance
- Violating code around them = -5 (they lose respect)

These values are defined in `config.yaml` under `trust.modifier_effects.HONOR_BOUND.instant_shifts`.

## Key Behavioral Notes

- Utterly predictable if you know the code
- Can be exploited through the code (get them to swear oath)
- Will not break word even to save loved ones
- Death before dishonor is literal, not metaphorical
- May require formal rituals (challenges, oaths)
- Once oath given, nothing can break it
- Respects others who live by codes (even enemies)
